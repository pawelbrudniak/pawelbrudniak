## 🔍 How I approach testing

I treat testing as a **process**, not a task:

1. Understand the system
2. Identify risk areas
3. Design realistic scenarios
4. Execute and observe behavior
5. Report defects with context
6. Investigate root cause

I focus on:
- real user flows
- system behavior
- failure analysis
